# Конвенции проекта

## Язык документации

Вся документация пишется на **русском языке**. Английский допускается только в:
- исходном коде (Nix, YAML, Shell);
- именах файлов и каталогов;
- названиях пакетов, сервисов и переменных;
- выводе программ.

## Структура каталогов

| Каталог | Назначение |
|---------|------------|
| `modules/system/` | Системные NixOS-модули (ядро, сеть, железо, сервисы, безопасность) |
| `modules/user/` | Пользовательские модули (shell, CLI, платформа) |
| `hosts/` | Конфигурации отдельных хостов (NixOS или Home-Manager) |
| `home/` | Общие Home-Manager фрагменты |
| `secrets/` | Зашифрованные SOPS-файлы (никогда не коммитятся в открытом виде) |
| `docs/` | Документация проекта |

## Именование файлов

- Модули: `snake_case` (например, `network.nix`, `zsh-macos.nix`).
- Конфигурации хостов: `configuration.nix`, `hardware-configuration.nix`, `home.nix`.
- Документация: `kebab-case` (например, `secrets-guide.md`, `ci.md`).
- Каталоги модулей: `snake_case` (например, `system/core/`, `user/shell/`).

## Формат документации

### Общие требования
- Файлы размещаются в `docs/`.
- Относительные ссылки между документами (`../hosts/ms-02.md`).
- Каждый файл содержит краткое вводное описание (1–2 предложения).
- Таблицы используются для структурированной информации.
- Блоки кода с указанием языка (` ```nix `, ` ```bash `).

### Шаблон модуля `docs/modules/`

```markdown
# Модуль `<category>/<name>`

**Путь:** `modules/<category>/<name>.nix`
**Назначение:** <одно предложение>
**Используется хостами:** <список или «все»>

## Описание
<2–4 предложения>

## Параметры
| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|-----------|

## Пример подключения
```nix
imports = [ ./path/to/module.nix ];
```

## Зависимости
- Secret'ы: ...
- Другие модули: ...

## См. также
- Ссылки на связанные документы
```

## Mermaid

Для диаграмм используется синтаксис Mermaid. Все диаграммы должны иметь fallback в виде ASCII-дерева или таблицы.

## Проверка

Документация проверяется через:
- `markdownlint docs/**/*.md` — форматирование;
- ручная проверка ссылок между файлами.
